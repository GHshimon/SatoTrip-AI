# スポットDB 全国ロールアウト計画（5か月・無料枠ペース）

作成日: 2026-07-12 / 前提: Google Places 無料枠（Enterprise 1,000回/月）を律速に、月約10県ペースで
47都道府県を約5か月で整備する。詳細な課金根拠は本ファイル末尾と `SPOT_FIELD_SPEC.md §5` 参照。

## ペース設計

- **律速**: Places Enterprise SKU 無料枠 = 1,000回/月（Place Details＝営業時間/評価取得に必須）。
  1県あたり Details 約35回 → **月約10県**が無料枠上限（現行マスク）。マスク最適化後は月約28県まで可能だが、
  本計画は安全側に**月10県ペース**で5か月を組む。
- **日次の副次律速**: Gemini 約250req/日（≈2〜3県/日）、YouTube 100検索/日（≈5県/日）。
  → 月内は**週2〜3県ずつ**に分散実行し、日次枠も超えないようにする。
- **月初リセット活用**: Places無料枠は毎月1日にリセット。各月は**その月の対象県だけ**を消化する。
- **1県あたり目安**: 検証OK（verified）30〜50スポット。品質優先（未検証・棄却は非公開）。

## 実行手順（1県あたり）

1. 管理画面 → スポット → 一括追加、または `POST /api/spots/bulk-add-by-prefecture`（管理者認証）。
   - `prefecture`: 対象県、`run_async: true`（バックグラウンドジョブ）
   - 既定の上限（20キーワード×3件）でよい。需要の高い県は `category` 指定で複数回に分けても可。
2. 完了後、`BulkAddResponse` の内訳を確認: `verified_count / needs_review_count / rejected_count` と
   `places_search_count / places_hit_count / details_call_count`。
3. `needs_review` は管理画面の承認キュー（Phase 2 で実装予定）で人手確認 → verified 化。
4. **月内の累計 Place Details 呼び出しが 900 に達したら自動で打ち切り**（Enterprise枠1,000の安全マージン）。
   これは**コード側で強制**される（月次予算ガード。`places_usage_service` が Details 呼び出しを月単位で
   永続集計し、`bulk_add_spots_by_prefecture` は上限到達時に新規実行を停止する）。閾値は
   `PLACES_MONTHLY_DETAILS_SOFT_LIMIT`。今月の使用量は管理APIの `GET /api/spots/places-usage` で確認可。
   無料枠は翌月1日にリセット（＝カウンタも新しい年月キーで自動リセット）。

## スケジュール登録の設定（①リマインダー方式・恒久ルーティン）

方針: **①リマインダー方式で開始 → 前提が整い次第②完全自動へ切替**（オーナー合意済み）。

恒久スケジューラ（週次ルーティン）に以下を登録する。※本セッションでは登録ツールの承認応答が
返らず未登録。ツール回復後にこの設定で登録すること（前提キー未設定のため急がない）。

- **名称**: SatoTrip スポットDB 全国ロールアウト（週次リマインダー・5か月）
- **起動**: 毎週月曜 09:00 JST（cron `0 0 * * 1` = 月曜 00:00 UTC）
- **セッション**: 毎回フレッシュセッション（`create_new_session_on_fire: true`）
- **通知**: push + email
- **発火プロンプト（要旨）**:
  1. CLAUDE.md / 本ファイル / SPOT_FIELD_SPEC.md を読む
  2. 本番DBの現状を確認し、優先順位で**未整備の上位2〜3県**を「今週の対象」に選ぶ
  3. 前提条件を確認（GOOGLE_MAPS_API_KEY / YouTube キー / 予算アラート / 管理者実行可否）
  4. **①既定=リマインダー**: 今週の対象県・実行手順・不足している前提をオーナーへ通知（自分では本番APIを叩かない＝課金しない）
  5. **②切替後のみ**: オーナーが「完全自動に切替」と明示し前提が全て整っている場合に限り、対象県の一括追加（run_async）を実行し内訳（verified/needs_review/rejected/details_call_count）を報告。月内 details 900回で打ち切り
  6. 47県すべて整備済みなら「ロールアウト完了。ルーティン停止可」と報告
- **②完全自動への切替条件**: OWNER_TASKS の Places新キー・YouTubeキー・予算アラートが完了し、
  週次セッションが本番管理APIを叩くための管理者認証手段が用意できたとき。切替時に上記プロンプトの
  「既定=リマインダー」を「既定=自動実行」へ変更する。
