## 0. まず「今やるべきか」

**結論: 方向性を固めて“動くモック”で握るのは今が適期。全画面の本適用は段階的に。**

機能面は重大バグ修正・新機能・E2E 検証・耐障害性強化で安定しました。土台が固まった今、見た目への投資は妥当です。ただし全画面の一括差し替えは早い。先に **前提作業**（Tailwind/React の CDN 依存 → Vite ビルド一本化＋トークン定義）を済ませ、**コア画面から段階移行**します。本提案は `main` と別ブランチに隔離しており、進行中の改修に影響しません。

## 1. コンセプト: 「旅の同人誌 × 地形図」

「引き寄せるリッチさ」を **旅行サイトにありがちな“豪華な写真の全面敷き”では実現しない**、というのが本提案の芯です。理由は 2 つ:

1. **写真依存は全国 × ニッチと相性が悪い** — 全都道府県・無数のニッチ趣味を高品質写真で埋めるのは非現実的。写真が無い/弱い所で一気に安っぽく崩れる。
2. **差別化にならない** — 美しい写真カードは競合と同じ土俵。SatoTrip の資産は写真ではなく **「趣味の軸 × 地元のインサイダー情報 × 旅路のデータ」**。

そこで写真の代わりに、**旅路そのものをコードで生成した美しいグラフィック**（等高線の地形・ルートの紋章・織物のように編まれるしおり）で “刺さるリッチさ” を作ります。世界観は **「地元の人が手作りした旅の同人誌」×「登山地図・地形図の精緻さ」**。

3 つの柱:
1. **趣味で検索** — 8 つの趣味が **和色 + 生成アイコン** を持ち、選ぶと動線・スポット・並びが変わる（意味＝色）。
2. **地元インサイダー** — 常連の“ひとこと”を **明朝（人の声）** で、看板のない穴場を可視化。
3. **旅路のデータを主役に** — 座標・順路・趣味の色から **「ルート紋章」** を生成。写真ではなく“あなたの旅そのもの”が絵になる。

## 2. 理論的裏付け（リサーチ要約）

本提案は感覚ではなく、以下の理論から各判断を導いています。

**視覚原則**
- **美的ユーザビリティ効果 / 50ms の第一印象** — ヒーローで世界観を一撃で伝える（hero-as-thesis）。
- **ゲシュタルト（近接・共通運命）** — 趣味カードのグリッド群化、ルート線で点を「旅」として繋ぐ。
- **F/Z 型視線** — 左上ブランド → 見出し → CTA の Z 動線。
- **色 = 情報（categorical color）** — 8 趣味を OKLCH で明度・彩度を揃えた **和色** に割当。装飾ではなく識別子。
- **タイポの役割分担** — **明朝（Zen Old Mincho 系）= 物語る声 / ゴシック（角ゴ系）= 道具・UI**。「人の声」と「操作」を書体で分離。CSP で webフォント不可のため **日本語システム明朝/ゴシックのスタック** で再現。

**モーション原則（安っぽさ回避の要）**
- **動きの主役は 1 画面 1 つ**、残りは opacity のみの脇役。全要素が同じ動きで浮く＝典型的な安さ。
- **イージングは強い減速 1 種に統一**（`cubic-bezier(.22,.61,.36,1)`）、バウンス禁止。
- **速度の階層** — 背景は数十秒周期、UI は 0.3–0.8s、マイクロは 0.15s。
- **動きに物語の根拠を** — 「等高線が流れる」「ルートが引かれる」「しおりが編まれる」。根拠のない動きは削る。
- **`prefers-reduced-motion` 完全対応** — 全アニメ停止、コンテンツは即時全表示。

**アンチパターン（意図的に回避）**: インディゴ一色の量産グラデ / Inter・Noto だけの無個性 / クリーム＋セリフ＋テラコッタの“AIっぽい”定番 / カード 3 枚横並び / 全中央寄せ / 絵文字アイコン / 全部 `rounded-lg`。

## 4. タイポグラフィ

- **見出し・人の声**: 明朝系 `"Hiragino Mincho ProN","Yu Mincho","Noto Serif JP",serif`
- **UI・データ・ラベル**: 角ゴ系 `"Hiragino Kaku Gothic ProN","Yu Gothic","Noto Sans JP",sans-serif`
- 数字揃えは `tabular-nums`、和文は `font-feature-settings:"palt"` で詰め、見出しは `text-wrap:balance`。
- webフォント CDN は CSP で不可 → **システムフォントスタックのみ**でフォールバック破綻を回避。

## 5. 生成グラフィック（写真の代替）— 本提案の目玉

すべて **単一 HTML・外部リソース 0・CSP セーフ**（インライン vanilla JS / Canvas2D / 生 WebGL / SVG feTurbulence）。

1. **動く等高線ヒーロー背景（WebGL）** — value noise + fbm + ドメインワープで、ゆっくり流れる **地形図の等高線**。`fract()` で等高線化、藍で描画。「里山の地形の上を旅する」情緒をコンセプト（地形図）そのままに表現。DPR 1.5 クランプ・`prefers-reduced-motion` で 1 フレーム静止。WebGL 不可時は CSS グラデにフォールバック。
2. **ルート紋章（Canvas 生成）** — 訪れた座標・順路・趣味の色から、コンパスの目盛り・等高線の反響・ルート線・方位星で **二つと無い紋章** を deterministic に生成。旅の記録でありしおりの表紙。
3. **趣味アイコン（Canvas 生成）** — 8 趣味それぞれに線画グリフ（ルート/豆/波/レンズ/鳥居/葉/器/窯）。hover で線が趣味色に発色。
4. **地形ミニマップ** — インサイダー節の sticky カードに等高線＋ルート。スクロールで説明ステップに同期。
5. **フィルムグレイン** — SVG `feTurbulence` を全面に薄く（opacity .05, blend mode）重ね、紙・記憶の質感を全体に統一。

## 6. モーション構成

| 箇所 | 演出 | 根拠 |
|---|---|---|
| ヒーロー | 等高線背景（60s 級の超低速）＋見出しのマスク・リビール（段階） | 「幕が上がる」。動きの主役は 1 つ |
| 本文 | `IntersectionObserver` の出現（80ms 階段）、一度きり | 読み進み＝旅の前進 |
| インサイダー | sticky 地図 × スクロール連動でステップ点灯／地図タグ切替 | 「旅程を辿る」scrollytelling |
| 待ち時間 | **「しおりが編まれる」織物ローダー**（縦＝経糸、横＝趣味色の緯糸が順に織られ、朱の杼が走る） | 生成の数十秒を世界観に転化。無機質スピナーを廃す |
| CTA/統計 | ホバーは `translateY(-2px)＋影` の最小、統計は将来カウントアップ | 安さの典型（跳ね/回転）を回避 |

`transform`/`opacity` のみでコンポジタ完結、`scroll` は passive + rAF 集約。

## 7. 段階移行プラン（本適用時）

1. **前提** — Tailwind/React の CDN 依存を排し Vite ビルドへ。トークンを CSS 変数 + TS 定義で一元化。
2. **基盤** — グレイン・和色トークン・書体スタック・reduced-motion を全体へ。
3. **コア画面から** — ①ランディング ②趣味検索（8 和色チップ）③プラン詳細（ルート紋章＋しおり）④生成待ち（織物ローダー）。
4. **段階検証** — 各画面ごとに E2E とライトハウス（LCP/CLS）を確認しつつ差し替え。

写真は「あれば添える」補助に降格し、**基幹の魅力は生成グラフィックが担う**ため、全国 × ニッチでも品質が崩れません。
